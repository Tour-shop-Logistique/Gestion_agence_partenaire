════════════════════════════════════════════════════════════════════════════════
   ⚠️ PROBLÈME IDENTIFIÉ
════════════════════════════════════════════════════════════════════════════════

AVANT :
   Lorsqu'un code QR était placé devant le scanner, il était scanné en continu
   (plusieurs fois par seconde), ce qui causait :
   
   ❌ Sons répétés en boucle
   ❌ Messages toast multiples
   ❌ Expérience utilisateur dégradée
   ❌ Performance impactée

════════════════════════════════════════════════════════════════════════════════
   ✅ SOLUTION IMPLÉMENTÉE
════════════════════════════════════════════════════════════════════════════════

1. 🕐 SYSTÈME DE COOLDOWN (2 SECONDES)
   ────────────────────────────────────
   • Chaque code scanné est mémorisé avec son timestamp
   • Si le même code est détecté dans les 2 secondes, il est ignoré
   • Après 2 secondes, le même code peut être scanné à nouveau

2. 📝 TRACKING DU DERNIER SCAN
   ────────────────────────────────────
   • Utilisation de useRef pour stocker :
     - Le code scanné (lastScannedRef.current.code)
     - L'horodatage du scan (lastScannedRef.current.timestamp)
   • Comparaison avant chaque nouveau scan

3. 🎨 FEEDBACK VISUEL
   ────────────────────────────────────
   • Affichage du code scanné pendant 2 secondes
   • Indicateur visuel avec animation (point vert clignotant)
   • Message clair "✓ Code scanné : XXX"

   ⏱️ TIMELINE D'UN SCAN
════════════════════════════════════════════════════════════════════════════════

T = 0s     │ 📷 Code QR détecté "COL001"
           │ ✅ Premier scan accepté
           │ 🔊 Son joué
           │ 💬 Toast affiché
           │ 🎨 Feedback visuel activé
           │
T = 0.5s   │ 📷 Code QR détecté "COL001" (encore devant caméra)
           │ ❌ IGNORÉ (cooldown actif)
           │
T = 1s     │ 📷 Code QR détecté "COL001"
           │ ❌ IGNORÉ (cooldown actif)
           │
T = 1.5s   │ 📷 Code QR détecté "COL001"
           │ ❌ IGNORÉ (cooldown actif)
           │
T = 2s     │ 🎨 Feedback visuel disparaît
           │
T = 2.5s   │ 📷 Code QR détecté "COL001"
           │ ✅ Scan accepté (cooldown expiré)
           │ 🔊 Son joué à nouveau
           │
T = 3s     │ 📷 Code QR différent "COL002"
           │ ✅ Scan accepté (code différent)

PARAMÈTRE ACTUEL : 2000ms (2 secondes)

Pour modifier la durée du cooldown, chercher dans QRScanner.jsx :

    if (lastScannedRef.current?.code === decodedText && 
        lastScannedRef.current?.timestamp && 
        (now - lastScannedRef.current.timestamp) < 2000) { // ← ICI
        return;
    }

RECOMMANDATIONS :
   • 1000ms (1s) : Trop court, risque de scans multiples
   • 2000ms (2s) : ✅ OPTIMAL - Balance parfaite
   • 3000ms (3s) : Acceptable mais peut sembler lent
   • 5000ms+ : Trop long, frustrant pour l'utilisateur
