════════════════════════════════════════════════════════════════════════════════
   🔄 FONCTION handleQRScan AMÉLIORÉE
════════════════════════════════════════════════════════════════════════════════

AVANT :
───────
• Scan → Ajout direct à la sélection
• Toast simple à chaque scan
• Pas de son
• Pas de vérification de doublon dans selectedCodes

APRÈS :
───────
• Scan → Vérification état (expédié / sélectionné / nouveau)
• Toast unique par colis (tracking avec scannedCodesRef)
• Sons différenciés selon le résultat
• Vérification anti-doublon stricte

FLUX DÉTAILLÉ :
───────────────

handleQRScan(scannedData) {
    
    1. RECHERCHE DU COLIS
       ├─ Par code_colis exact
       ├─ Par code_colis partiel
       └─ Par ID d'expédition
    
    2. COLIS TROUVÉ ?
       │
       OUI ─→ 3. VÉRIFIER ÉTAT
       │      │
       │      ├─ is_sent = true (Déjà expédié)
       │      │  ├─ Déjà scanné ? → Rien faire
       │      │  └─ Nouveau scan ? → Son warning + Toast info + Tracking
       │      │
       │      └─ is_sent = false (Disponible)
       │         │
       │         ├─ Déjà dans selectedCodes ?
       │         │  ├─ Déjà scanné ? → Rien faire
       │         │  └─ Nouveau scan ? → Son warning + Toast "déjà sélectionné" + Tracking
       │         │
       │         └─ Pas dans selectedCodes ?
       │            ├─ Ajouter à selectedCodes
       │            ├─ Son succès 🔊
       │            ├─ Toast succès 💬
       │            ├─ Tracking ✓
       │            └─ Scroll vers le colis 📜
       │
       NON ─→ Son erreur + Toast erreur (toujours affiché)
}

   🎯 SCÉNARIOS D'UTILISATION
════════════════════════════════════════════════════════════════════════════════

SCÉNARIO 1 : Scan d'un nouveau colis disponible
────────────────────────────────────────────────
   📷 Scan "COL001"
   ✅ Colis trouvé et disponible (is_sent = false)
   ✅ Pas dans selectedCodes
   ➡️  Ajouté à la sélection
   🔊 Son de succès (2 notes)
   💬 Toast : "Colis COL001 sélectionné !"
   📜 Scroll vers le colis
   
   État : selectedCodes = ["COL001"]

SCÉNARIO 2 : Re-scan du même colis
───────────────────────────────────
   📷 Scan "COL001" (après cooldown QR de 2s)
   ✅ Colis trouvé et disponible
   ⚠️  Déjà dans selectedCodes !
   ➡️  NON ajouté (évite doublon)
   🔊 Son d'avertissement
   💬 Toast : "Le colis COL001 est déjà sélectionné." (une fois)
   
   État : selectedCodes = ["COL001"] (inchangé)

SCÉNARIO 3 : Scan d'un colis déjà expédié
──────────────────────────────────────────
   📷 Scan "COL999"
   ✅ Colis trouvé
   ⚠️  Déjà expédié (is_sent = true)
   ➡️  NON ajouté
   🔊 Son d'avertissement
   💬 Toast : "Le colis COL999 a déjà été expédié vers l'entrepôt." (une fois)
   
   État : selectedCodes = [] (inchangé)

SCÉNARIO 4 : Scan d'un colis inexistant
────────────────────────────────────────
   📷 Scan "UNKNOWN"
   ❌ Colis non trouvé
   ➡️  Aucune action
   🔊 Son d'erreur (note grave)
   💬 Toast : "Aucun colis trouvé avec le code scanné : UNKNOWN"

SCÉNARIO 5 : Envoi des colis sélectionnés
──────────────────────────────────────────
   🖱️  Clic "Envoyer à l'entrepôt"
   📤 Envoi à l'API
   ✅ Succès
   
   ACTIONS :
   ├─ 🔊 Son de succès
   ├─ 💬 Toast de confirmation
   ├─ 🔄 Reset selectedCodes = []
   └─ 🔄 Reset scannedCodesRef = Set()
